# Purpose

This document defines the exercise requirements for every chapter in Volume I.

Its purpose is to ensure that every chapter provides sufficient practice opportunities for readers to master the concepts being taught.

---

# Exercise Philosophy

Readers do not become programmers by reading.

Readers become programmers by writing code.

Therefore, every chapter must contain enough exercises to reinforce the concepts covered.

# Difficulty Levels

Exercises should gradually increase in difficulty.

### Level 1 — Beginner

Simple recall questions.

### Level 2 — Understanding

Readers explain concepts.

### Level 3 — Application

Readers write small programs.

### Level 4 — Analysis

Readers debug existing code.

### Level 5 — Challenge

Readers solve complete programming problems.

---

# Mini Project Requirements

Every chapter should end with one practical mini project.

Each project must include:

- Objective
- Requirements
- Concepts Covered
- Expected Output
- Extension Challenge

---

# Chapter Challenge Requirements

Each chapter concludes with one larger programming task.

The challenge should combine multiple concepts from the chapter and encourage problem-solving.

---

# Success Criteria

Volume I exercise development is complete when:

- All exercises are written.
- All code examples have been tested.
- Every answer has been verified.
- Difficulty levels are balanced.
- Mini projects have been completed.
- Chapter challenges have been reviewed.
