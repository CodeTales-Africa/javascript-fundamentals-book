# Purpose

This document defines the illustration strategy for **Cracking JavaScript: Volume I**.

Figures should improve understanding—not decorate the book. Every figure must have a clear educational purpose.

---

# Figure Objectives

Every illustration should help readers:

- Visualize abstract concepts.
- Understand JavaScript execution.
- Follow programming workflows.
- Learn through diagrams.
- Connect theory with practice.

---

# Figure Categories

| Figure Type | Description |
|--------------|-------------|
| Concept Diagram | Explains programming concepts visually |
| Flowchart | Shows execution flow or decision making |
| Syntax Diagram | Breaks down JavaScript syntax |
| Browser Screenshot | Demonstrates browser tools |
| VS Code Screenshot | Demonstrates development environment |
| Code Walkthrough | Explains code step by step |
| Table | Compares concepts |
| Timeline | Shows historical events |
| Architecture Diagram | Explains how JavaScript works |
| Process Diagram | Shows a sequence of operations |

# Figure Numbering Standard

Every figure should follow this format:

```
Figure ChapterNumber.FigureNumber
```

Examples:

```
Figure 1.1

Figure 1.2

Figure 2.1

Figure 5.4

Figure 8.7
```

---

# Figure Caption Standard

Every figure must include:

- Figure number
- Short descriptive title
- One-sentence explanation

Example:

```
Figure 8.3

Creating an Array Using Array Literal Syntax

This figure illustrates how array elements are stored in sequential indexes.
```

---

# Figure Placement Rules

Figures should appear:

- Immediately after the related explanation.
- Before the practical example, where appropriate.
- On the same page whenever possible.
- Close to the text that references them.

---

# Figure Design Guidelines

All figures should:

- Use consistent fonts.
- Use consistent colors.
- Be easy to read.
- Have sufficient spacing.
- Include meaningful labels.
- Avoid unnecessary decoration.
- Be suitable for both print and digital versions.

---

# Screenshot Standards

Screenshots should:

- Use Visual Studio Code.
- Use Google Chrome Developer Tools where applicable.
- Be high resolution.
- Avoid personal information.
- Show only relevant content.

---

# Diagram Standards

Concept diagrams should:

- Be simple.
- Use arrows to indicate flow.
- Clearly label all components.
- Focus on one idea at a time.

---

# Tables

Tables should be used for:

- Comparisons
- Feature summaries
- Operator precedence
- Method references
- Syntax differences

---

# Figure Checklist

Every figure should satisfy the following:

- [ ] Figure number assigned
- [ ] Caption written
- [ ] Referenced in the text
- [ ] High resolution
- [ ] Consistent formatting
- [ ] Educational value confirmed
