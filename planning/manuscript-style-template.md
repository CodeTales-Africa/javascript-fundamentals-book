# Purpose

This document defines the official manuscript formatting standards for **Cracking JavaScript: Volume I**.

Every chapter, example, figure, table, exercise, and project must follow these standards to ensure consistency throughout the book.

# Paragraph Style

- Use short paragraphs.
- Explain one concept at a time.
- Avoid unnecessary jargon.
- Introduce technical terms before using them.
- Prefer active voice.

---

# Code Block Style

All JavaScript code must use fenced code blocks.

Example

```javascript
let name = "John";

console.log(name);
```

Rules

- Indent with 4 spaces.
- Use meaningful variable names.
- Keep examples concise.
- Test all code before publishing.
- Use modern JavaScript (ES6+).

---

# Code Comments

Use comments only when they improve understanding.

Example

```javascript
// Store the user's age
let age = 25;

// Display the age
console.log(age);
```

Avoid unnecessary comments.

# Grammar Standards

Use:

- American English spelling
- Present tense where appropriate
- Active voice
- Consistent terminology

Examples

Correct

```
JavaScript stores data in variables.
```

Avoid

```
Data is stored in variables by JavaScript.
```
