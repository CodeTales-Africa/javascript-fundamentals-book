# Code of Conduct

## Our Commitment

At **CodeTales Africa**, we are committed to creating a welcoming, inclusive, and respectful environment for everyone, regardless of experience level, background, identity, or nationality.

Our goal is to build a positive learning community where developers can collaborate, ask questions, share ideas, and grow together.

We believe that everyone deserves to participate in an environment free from harassment, discrimination, and intimidation.

---

# Our Standards

Examples of behavior that contribute to a positive community include:

- Being respectful and considerate.
- Using welcoming and inclusive language.
- Being patient with beginners.
- Giving constructive feedback.
- Accepting constructive criticism gracefully.
- Helping others learn and grow.
- Respecting different viewpoints and experiences.
- Collaborating professionally.
- Sharing knowledge openly.
- Encouraging curiosity and continuous learning.

---

# Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment or bullying.
- Hate speech.
- Discrimination of any kind.
- Personal attacks or insults.
- Threatening or intimidating language.
- Trolling or deliberately disruptive behavior.
- Publishing another person's private information without permission.
- Sexual or inappropriate comments.
- Spam or promotional abuse.
- Any behavior that makes others feel unsafe or unwelcome.

---

# Respect for Beginners

This project is designed to help people learn JavaScript.

Questions that may seem simple or repetitive are welcome.

Everyone starts somewhere, and we encourage experienced developers to help beginners with patience and kindness.

---

# Community Expectations

As a member of this community, you are expected to:

- Treat everyone with respect.
- Keep discussions professional.
- Focus on solving problems rather than criticizing people.
- Share knowledge honestly.
- Encourage learning and collaboration.
- Follow GitHub best practices when contributing.

---

# Contributing

Before submitting contributions, please read the following documents:

- `CONTRIBUTING.md`
- `README.md`
- `LICENSE`

When contributing code:

- Write clean and readable JavaScript.
- Follow the project's coding standards.
- Test your code before submitting.
- Use meaningful commit messages.
- Keep pull requests focused and well documented.

---

# Reporting Issues

If you experience or witness unacceptable behavior, please open a GitHub Issue or contact the project maintainer.

Please include:

- A description of the incident.
- Any relevant screenshots or links.
- Steps to reproduce the issue (if applicable).

All reports will be reviewed fairly and respectfully.

---

# Enforcement

Project maintainers are responsible for ensuring that this Code of Conduct is followed.

Maintainers may:

- Remove inappropriate comments.
- Close abusive issues or pull requests.
- Warn contributors.
- Temporarily restrict participation.
- Permanently ban users who repeatedly violate this Code of Conduct.

Enforcement decisions will be made with fairness and the best interests of the community in mind.

---

# Scope

This Code of Conduct applies to:

- GitHub Issues
- Pull Requests
- Discussions
- Project documentation
- Community interactions related to this project
