# Development Workflow

Follow these steps when contributing:

## 1. Fork the Repository

Create your own copy of the repository on GitHub.

---

## 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/javascript-fundamentals-book.git
```

---

## 3. Create a New Branch

Always create a separate branch for your work.

```bash
git checkout -b feature/add-array-examples
```

Use descriptive branch names such as:

- feature/add-dom-examples
- fix/chapter-05-typos
- docs/update-readme
- improvement/chapter-12

---

## 4. Make Your Changes

Examples include:

- Improve documentation
- Add JavaScript examples
- Fix bugs
- Add exercises
- Improve explanations

---

## 5. Test Your Changes

Before submitting:

- Ensure all JavaScript examples run correctly.
- Check for spelling and grammar mistakes.
- Verify links are working.
- Confirm formatting is consistent.

---

## 6. Commit Your Changes

Write clear and meaningful commit messages.

Good examples:

```text
Add array iteration examples

Fix typo in Chapter 5

Improve DOM event explanation

Add exercise solutions for Chapter 12
```

Avoid messages like:

```text
Update

Stuff

Changes

Fix
```

---

## 7. Push Your Branch

```bash
git push origin feature/add-array-examples
```

---

## 8. Open a Pull Request

Describe:

- What you changed
- Why you changed it
- Screenshots (if applicable)
- Related issues

# Coding Standards

Please follow these guidelines:

## JavaScript

- Use modern JavaScript (ES6+)
- Prefer `const` over `let` whenever possible.
- Avoid using `var`.
- Use descriptive variable names.
- Write readable code.
- Keep functions focused on one task.
- Use meaningful comments when necessary.

Example:

```javascript
const calculateArea = (width, height) => {
    return width * height;
};
```

# Code Examples

Every example should:

- Be simple
- Be easy to understand
- Be well commented when necessary
- Demonstrate one concept at a time

Good example:

```javascript
const message = 'Hello, JavaScript!';

console.log(message);
```
